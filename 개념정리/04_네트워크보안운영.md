# 네트워크 보안 운영 — 개념 정리

> 학습모듈: 네트워크 보안 솔루션을 운영·개선하고, 신규 위협에 대응하며, 솔루션 업데이트를 적용한다.

---

## 학습 1. 네트워크 보안 솔루션 운영하기

### 1-1. 네트워크 보안 요구사항 분석

#### 네트워크 보안 요구사항 (보안 4요소 측면)
- 기밀성 측면 : 허용된 트래픽만 통신 허용, 유해 IP·URL·악성 패킷 차단, 중요 정보 송수신 시 암호화, 비인가 사용자 접근 불가.
- 무결성 측면 : 송·수신자 간 전송 과정에서 패킷의 위조·변조가 발생하지 않아야 함.
- 가용성 측면 : 패킷이 적절한 시간에 도달하도록 대역폭 보장, DoS 등 비정상 패킷 증가 상황에서도 정상 서비스 제공.
- 부인 방지 측면 : 종단 간 주소(IP)와 통신 포트(port)가 식별되고, 종단 간 메시지 전송을 보장.

#### 네트워크 장비 보안 요구사항
- 식별 및 인증 : 디폴트 관리자 패스워드 설정 금지, 안전한 패스워드 설정, 인증 실패 횟수 초과 시 대응, 패스워드 마스킹, 인증 정보 평문 저장 금지.
- 자체 보안 관리 : ACL 규칙 트래픽 제어, SNMP 모니터링·관리, 관리 콘솔 접속 IP 제한, 주기적 자체 시험, 설정값·실행 코드 무결성 검사, 펌웨어 무결성 검증.
- 세션 관리 및 접근 통제 : 무활동 시 세션 잠금/종료, 동시 원격 세션 제한, 웹 UI 인증 정보 재사용 방지, 사용자·그룹별 접근 권한 설정, 운영 모드 제공.
- 전송 데이터 보호 : 원격 관리 콘솔·외부 서버 연동 시 SSL/TLS·HTTPS·SSH·IPSEC 등 사용.
- 감사 기록 : 감사 시작·종료, 인증 성공·실패 기록, 설정 변경·보안 기능 수행 내역, 사건 발생 일시·유형 제공, 감사 증적 용량 초과 시 처리방안.

#### 네트워크 보안 취약점 유형 (진단 대상 기반)
- 네트워크 기반 취약점 : 전체 네트워크를 대상으로 진단을 수행하며 네트워크 관리자가 먼저 수행해야 하는 진단으로, 높은 위험을 가진 취약점을 즉시 파악하고 잘못된 방화벽 설정·취약한 정보시스템을 쉽게 탐지(Nexpose·Nessus·OpenVas 등 도구 사용).
- 호스트 기반 취약점 : 특정 호스트를 대상으로 점검을 수행하며 클라이언트-서버 모델에서 클라이언트가 스캔 후 관리자에게 결과를 전송, 분석 결과가 점검 방법에 의존적.

#### 네트워크 장비 보안 기능 상세 요구사항
- 안전한 비밀 번호 설정 기능 : 비밀번호는 9자리 이상 입력 가능해야 하며 숫자·영문자 대소문자·특수문자의 조합으로 구성할 수 있어야 함.
- 인증 실패 대응 기능 : 지정된 횟수(기본값 5회 이하) 이상 인증 실패 시 일정 시간(기본값 5분 이상) 장비 접속을 제한하는 기능.
- 인증 데이터 보호 기능 : 비밀번호를 대칭 키 암호·해시 함수 등으로 안전하게 저장하고, 장비에 저장된 모든 비밀 정보(비밀번호·사전 공유 키·대칭 키·개인 키 등)를 읽거나 유추할 수 없어야 함.
- 암호 사용(보안 강도) : 암호화 및 해시 알고리즘의 보안 강도는 112bit급 이상(해시 SHA-224/256 등, 대칭 키 SEED·ARIA, 공개 키 RSA 2048, 전자 서명 RSA-PSS·ECDSA 등)을 만족해야 함.
- ACL 기능(장비 계층별) : L3 장비는 IP 주소(Source·Destination), L4 장비는 IP 주소+포트, L5 이상 장비는 IP 주소+포트+프로토콜로 트래픽을 제어.
- 원격 관리 기능 : 원격 관리 서비스(SSH·HTTP/HTTPS·FTP·TELNET·SNMP 등)는 디폴트 비활성화하며, SNMP 지원 시 V3 이상을 지원해야 함.
- 오류 검사 기능 : 장비 구동 시(Power On) CPU·메모리·플래시·네트워크 인터페이스 등 주요 하드웨어와 펌웨어 로딩 후 주요 프로세스의 오류를 확인하는 자체 검사 기능.
- 안전한 암호 통신 프로토콜 : 원격 접속 시 신뢰된 채널을 제공하며 TLS는 1.2(RFC 5246) 이상, SSH는 v2(RFC 4251~4254) 이상을 지원해야 함.

### 1-2. 네트워크 보안 솔루션 운영

#### 네트워크 개념
- 통신의 3개 에이전트 : 애플리케이션(전자메일·파일 다운로드 등), 컴퓨터, 네트워크.
- 네트워크 프로토콜 : 2대 이상의 컴퓨터 통신을 위한 통신 규약. 대표적으로 ISO에서 개발한 OSI(Open System Interconnection) 모델.

**OSI 7 계층** (물리~네트워크: 하위 계층 / 전송~응용: 상위 계층)
- 응용 계층(application) : 사용자가 OSI 환경에 접근하여 상호작용. 통신에 사용할 프로토콜 타입을 결정하고 파일 전송·DB 접근·전자메일 등 프로토콜 제공.
- 표현 계층(presentation) : 데이터 타입을 전송 형식과 무관한 구문(syntax)으로 변환하고 암호화 수행.
- 세션 계층(session) : 애플리케이션·프로세스 간 접속 설정·관리·종료 제어.
- 전송 계층(transport) : 종단 간(end-to-end) 데이터 전송에 신뢰성을 제공. 에러 복구·흐름 제어.
- 네트워크 계층(network) : 경로 선택·중계 기능 수행, 데이터 크기를 적절히 분할(하나의 전송 링크 관리).
- 데이터 링크 계층(data link) : 물리적 링크 간 신뢰성 있는 정보 제공. 프레임 단위 전송, 순차 번호로 에러 검사·순서화.
- 물리 계층(physical) : 비트 레벨 통신. 전송 매체 간 인터페이스의 기계적·전기적 기능 정의.

#### 네트워크의 종류 (규모·전송 거리에 따라)
- 근거리 통신망(LAN) : 한 건물 또는 수 km 이내, 100Mbps 이내. 단일 기관 소유로 고속·저에러율. (이더넷-CSMA/CD, 토큰링, FDDI-광섬유 등)
- 도시 간 통신망(MAN) : LAN들을 연결한 LAN보다 크고 WAN보다 작은 도시 규모 네트워크.
- 광대역 통신망(WAN) : 국가·대륙 등 넓은 지역을 연결하는 원거리 통신망.
- 개인 통신망(PAN) : 10m 범위의 개인 영역 IT 장치 간 네트워크.
- 부가 가치 통신망(VAN) : 정보에 부가 가치를 첨가하여 불특정 다수에게 제공하는 네트워크.

#### 네트워크 보안 솔루션 종류와 개념
- 개념 : 네트워크상의 데이터·기기에 대한 기밀성·무결성·가용성을 향상하기 위한 솔루션.
- 방화벽(Firewall) : 외부→내부 접근 시 반드시 방화벽을 통과하도록 하여 내부 자원·정보를 보호하는 시스템. 관리자가 허용하는 패킷만 통과. 구성 요소: 네트워크 정책, 사용자 인증 시스템, 패킷 필터링, 응용 계층 게이트웨이.
  - 스크리닝 라우터 : 프로토콜 형태·송수신지 주소·포트 번호를 분석해 진입을 허가/거절하는 라우터. OSI 3·4계층(IP·TCP·UDP 헤더)에서 동작.
  - 베스천 호스트 : 보호된 네트워크에서 유일하게 외부 공격에 노출된 시스템. 내·외부 네트워크 간 게이트웨이 역할로 철저한 감시 필요.
- 가상 사설 네트워크(VPN) : 정보를 암호화하여 공중망으로 전송, 가로채기를 당해도 데이터 유출을 차단하는 솔루션.
- 침입 탐지 시스템(IDS) : 공격 형태를 분석해 사전 규칙(signature)을 만들고, 이상 패킷 유입 시 실시간 탐지·대응.
  - 기초 자료 종류에 따른 분류 : 단일 호스트 기반, 다중 호스트 기반, 네트워크 기반.
  - 침입 행위 기준에 따른 분류 : 정상적인 행위의 탐지(프로파일 기준 이탈 탐지), 잘못된 행위의 탐지(알려진 취약점 공격 프로파일 탐지).
- 침입 방지 시스템(IPS) : IDS에 방지 기능까지 자동 수행. 트래픽 모니터링으로 유해 트래픽 차단·세션 종료 등 능동적으로 네트워크를 보호.

#### LAN 세부 종류
- 이더넷(ethernet) : 제어 방식으로 CSMA/CD 표준안을 사용, 동축 케이블과 LAN 카드가 필요하며 노드 수가 많으면 전체 성능 저하 발생.
- 토큰링(token-ring) : 16Mbps 이하의 고속 전송이 가능하고 노드 수가 많아도 전송 속도 차이가 없으며, 리피터를 사용해 장거리 전송 시 에러가 없음.
- FDDI(fiber distributed data interface) : ANSI가 표준화한 네트워크 규격으로 광섬유 케이블을 사용하며 100Mbps의 고속 전송이 가능.

#### 네트워크 보안 운영 수립 절차
- 네트워크 위험 분석 : 대형 네트워크는 전체 운영 차원의 보안 수준 결정에 앞서 위험 분석을 수행하고, 개별 시스템·네트워크 관리자는 추가 보안 수준 결정을 위해 위험 분석을 수행.
- 의견 수렴 : 효과적인 보안 방침은 조직 관리층과 해당 방침을 사용할 조직원들의 의견을 수렴해 수립할 때 효과를 발휘.
- 보안 역할 및 책임 정의 : 네트워크·분산 환경에서는 보안 책임이 사용자 전체에 있으므로 관리자·사용자 차원의 역할과 책임을 정의하고 관련 교육을 시행.
- 보안 정책 정의 : 자산이 외부에 노출될 수 있음을 인식하고 네트워크 특성에 적합한 보안 정책을 정의(공격 발생 시 대응 방안 포함).

#### 공개 서버 및 원격 운영 보안
- DMZ(Demilitarized Zone) : 웹 서버 등 공개 서버를 내부 네트워크와 분리해 설치하는 영역으로, 침입 차단 시스템 등 보안 솔루션을 통해 보호.
- 원격 운영 보안 대책 : 원격지 네트워크 관리는 원칙적으로 금지하되 부득이한 경우 정보보호 최고 책임자 승인·접속 단말 및 사용자 인증·한시적 접근 권한 부여·전송 구간 암호화(VPN)·접속 단말 보안·지속적인 모니터링이 필요.

## 학습 2. 네트워크 보안 솔루션 운영 개선하기

### 2-1. 네트워크 보안 솔루션 운영 개선 방안 수립

#### 네트워크 보안 솔루션 개선 계획 절차
- 개념 : 다양한 보안 솔루션 운영 시 운영 인력 전문성 결여 등으로 취약점이 존재하므로 이를 보완할 개선 계획이 필요.
- 절차 : ① 네트워크 구성 분석(구성 요소·운영 중 솔루션 도출) → ② 보안 위협 도출(위협 타깃 설정, 위협 분석 모델링) → ③ 보안 취약점 진단(점검 항목 개발·진단) → ④ 개선 방안 수립(솔루션 패치 적용 등 조치·계획).

#### 네트워크 위협 및 취약점 진단 유형
- 네트워크 진단 : 원격 접속·네트워크 침입 흔적 등의 내용으로 네트워크 보안 취약점을 체계적으로 진단하여 종합적인 보안 대책을 수립.
- 모의 해킹 진단 : 외부에서 웹 환경을 이용해 내부망 공격을 시도하여 침투 가능성·안전성을, 내부에서는 파일명 변경 등을 통해 유출 가능성·안정성을 테스트하는 취약점 진단 기법.

#### 네트워크 보안 솔루션 점검 체크리스트
- 솔루션 SW/HW 관리 : 솔루션 인증 여부, 최신 패치 적용 여부, 불필요한 서비스 제거.
- 계정 관리 : default password 변경, 관리자 패스워드 주기적 변경.
- 네트워크 원격 관리 : 관리자 원격 관리 접근 통제, 원격 관리 시 idle 발생 시 접속 종료.
- 정책 관리 : 정책 중복 점검, 정책 변경 통제 절차, 변경 요청 룰의 기술적 검토, 침입 탐지 정책 검토.
- 로그 관리 : 로그 데이터의 저장, 정기적인 로그 점검.

### 2-2. 네트워크 보안 솔루션 운영 개선

#### 네트워크 보안 취약점 분석 절차 (7단계)
- 대상 시스템 결정 : 목표 시스템의 IP를 확인.
- 가동 시스템 탐색 : 핑거프린팅 기술로 살아 있는 호스트를 탐색.
- 서비스 목록 탐색 : 살아 있는 호스트에 포트 스캐닝 수행(취약 포트·서비스 식별, 가장 중요한 단계).
- 서비스 인식 : 열린 포트의 서비스를 시그니처 응답값 분석으로 확인.
- 애플리케이션 확인 : 서비스의 유형·벤더를 정확히 파악(취약점 진단 시 타 앱 영향·충돌 확인).
- 네트워크 취약점 확인 : 액티브 구성 조사 후 취약점 진단 및 커스텀 공격 실행.
- 발견된 취약점 보고 : 위험도 여부와 보안 대책을 기술한 보고서 작성.

#### 네트워크 보안 취약점 진단에 사용되는 포트
- FTP(21) : File Transfer Protocol. FTP 연결 시 인증과 컨트롤을 위한 포트.
- Telnet(23) : 원격지 서버의 실행 서비스.
- SMTP(25) : Simple Message Transfer Protocol. 메일을 보낼 때 사용하는 서비스.
- DNS(53) : Domain Name Service. 이름을 해석하는 데 사용하는 서비스.
- TFTP(69) : Trivial File Transfer Protocol. 인증이 존재하지 않는 단순한 파일 전송에 사용되는 서비스.
- HTTP(80) : Hyper Text Transfer Protocol. 웹 서비스.
- POP3(110) : Post Office Protocol. 메일 서버로 전송된 메일을 읽을 때 사용하는 서비스.
- RPC(111) : Remote Procedure Call. 원격에서 서버의 프로세스를 실행할 수 있게 한 서비스.
- NetBIOS(138) : Network Basic Input Output Service. 윈도우에서 파일을 공유하기 위한 서비스.
- IMAP(143) : Internet Message Access Protocol. POP3와 같으나 메일을 읽은 후에도 메일이 서버에 남는 것이 다름.
- SNMP(161) : Simple Network Management Protocol. 네트워크 관리와 모니터링을 위한 서비스.

#### 네트워크 보안 취약점 분석·평가
- 보안 권고문 : 네트워크 장비·애플리케이션 제조 업체나 각 나라별 CERT가 정기적으로 발표하는, 잘못된 환경설정·버그 등으로 발생 가능한 취약점 정보.
- 네트워크 보안 취약점 분석·평가의 목적 : 현재 네트워크 보안 노출 수준 점검, TCP/IP 정보시스템의 주요 보안 취약점 식별, 호스트 기반 보안 개선·발전, 네트워크·호스트 보안정책 적합성 확인, 위험 감소를 위한 기술적 보안 대책 수립.

#### 네트워크 보안 취약점 점검 위치
- 네트워크 구조 식별 : Ping·포트 스캔으로 동작 중인 컴퓨터·라우터·스위치·방화벽 등을 확인하고 실제 구성과 비인가 장비·시스템·접점을 탐지.
- 애플리케이션(서비스) 스캐닝 : 포트 스캐닝·배너 가로채기(banner grabbing) 기법으로 사용 중이거나 활성화된 서비스 통신 포트를 확인.
- 네트워크 취약점 점검 : 스캐닝·데이터 및 패스워드 유출·스푸핑·비인가 접근·DoS 등 기술적 위협과 사회 공학·물리적 비인가 접근 등 비기술적 위협을 사전에 발견(구조 식별·애플리케이션 스캐닝 포함).
- 시스템 취약점 점검 : 점검 대상 시스템에 에이전트를 설치해 실행 중인 정보시스템 서비스의 보안 취약점을 상세하게 파악.

## 학습 3. 네트워크 보안 신규 위협 대응하기

### 3-1. 네트워크 보안 위협 차단

#### 네트워크 공격 유형
- 서비스 거부(DoS) : 목표물을 장애 상태로 만들어 정상 서비스 요청을 거부하게 함(연결 시도 폭주 등).
- 분산 서비스 거부(DDoS) : 여러 장소에서 분산된 다수의 공격 컴퓨터로 목표물을 공격.
- SYN 플러드 공격 : 미완성 연결 요청(SYN 패킷)으로 가득 채워 정당한 연결을 처리하지 못하게 함(CPU·메모리·LAN 카드 이용률 급증).
- UDP 플러드 공격 : UDP 패킷을 계속 보내 시스템 속도를 저하시키고 정당한 연결을 막음(포트 53 DNS 플러딩이 전형).
- 포트 스캔 공격 : 이용 가능한 서비스 종류를 조사하기 위해 서로 다른 포트로 패킷을 보내 응답을 기다림.
- 죽음의 핑(ping of death) : 규정보다 긴 ICMP 패킷으로 DoS·시스템 파괴·고착·리부팅 유발.
- IP 스푸핑 : 유효한 클라이언트 IP·이메일·사용자 ID를 위조해 방화벽을 통과(컴퓨터 간 신뢰 관계 악용).
- 랜드 공격(land attack) : SYN 공격+IP 스푸핑. 목적지·발신지 IP를 모두 희생자 IP로 위장한 SYN 패킷으로 빈 커넥션을 양산하여 DoS 유발.
- 티어 드롭 공격(tear drop) : IP 패킷 재조립을 악용. 오프셋이 겹치도록 조작하여 재조립 시 문제 발생.
- 핑 스캔(ping scan) : ICMP 에코 요청을 여러 목적지로 보내 응답을 기다려 목표 가능 IP를 탐색.

#### 방화벽의 한계 및 내·외부 네트워크 점검
- 방화벽의 한계 : 내부 사용자가 디스크·USB 등 저장매체로 정보를 유출하는 침해, 시리얼 통신을 이용한 비인가 접근, 바이러스 감염, 내부 사용자의 악의적 행동은 차단할 수 없음.
- 외부 네트워크 점검 : 공격자 입장에서 외부에서 보안점검을 실시해 라우터·방화벽의 rule 설정 오류, 잘못된 내부 보안정책, 담당자 실수 등으로 인한 무방비 상태를 발견·대처.
- 내부 네트워크 점검 : 사용자 인증을 거치지 않고 내부 시스템에 접근하게 하는 백도어·방화벽 우회경로 존재 여부와 비인가자의 중요 시스템 접근 가능 여부를 점검해 내부 잠재 위험을 사전 감소.

### 3-2. 복구 및 문서화

#### 네트워크 보안 솔루션 관리 및 보호
- 보안 취약성 진단의 개념 : 정보시스템의 보안상 문제점을 파악하고 내·외부 침해로 발생 가능한 위험 수준을 진단하는 활동.
- 보안 취약성 진단의 목적 : 안티바이러스(사후 조치)와 달리 사전에 침해사고를 예방하는 적극적 보안 행위. 가용성·무결성을 보장하는 보안 운영 기법.
- 보안 취약성 진단 및 조치 : 운영 환경에 맞게 진단 주기를 적정 수준으로 조정해 지속 관리.
- 위협 인지 및 대응 : 단순 공격 탐지가 아니라 취약성을 파악한 후 공격을 탐지하고 영향분석(impact analysis) 수행.
- 최근 공격 경향 분석 : 최신 보안 취약성과 공격 여부를 분석하고, 자사 시스템 존재 여부·탐지·대응 방법을 파악.

#### 정보보호 관리체계
- 개념 : 일회성·단편적 정보보호에서 벗어나 조직적·종합적 정보보호 대책을 수립해 관리 수준 전반을 향상시키는 체계.
- ISO 27001 : 정보보호 관리를 경영 관리 시스템의 한 부분으로 인식하고, 위험관리 기반으로 정보보호의 수립·구현·운영·검토·개선을 하는 시스템. ISO27001:2022 기준 4개 도메인(조직 37·사람 8·물리적 14·기술적 34) 93개 통제 항목.
- ISMS-P 인증 : 인증 심사 필수 항목으로, 위협·취약성 변화에 대응하기 위해 지속 유지되는 순환주기 형태.
  - 정책 수립 단계 : 상위 수준 정보보호 정책 수립, 책임 설정, 법적·규제 요건과 위험 관리 기술.
  - 범위 설정 단계 : ISMS 범위 내 정보 자산 식별·목록화(형태·소유자·관리자 등).
  - 위험 관리 단계 : 위험 관리 전략·계획 수립, 위협·취약성·위험 식별 및 영향 분석.
  - 구현 단계 : 정보보호 계획에 따라 대책을 구현하고 교육·훈련 진행.
  - 사후 관리 단계 : 상시 모니터링, 정기 내부 감사로 정책 준수 확인 및 개선.

#### ISMS-P 인증 기준 영역
- ISMS-P 인증 기준 3대 영역 : 관리체계 수립 및 운영, 보호 대책 요구사항, 개인정보 처리 단계별 요구사항.
- 관리체계 수립 및 운영 영역 : 관리체계 기반 마련, 위험 관리, 관리체계 운영, 관리체계 점검 및 개선으로 구성.
- 보호 대책 요구사항 영역 : 정책·조직·자산 관리, 인적 보안, 외부자 보안, 물리 보안, 인증 및 권한관리, 접근통제, 암호화 적용, 정보시스템 도입·개발 보안, 시스템·서비스 운영/보안관리, 사고 예방 및 대응, 재해 복구로 구성.
- 개인정보 처리 단계별 요구사항 영역 : 개인정보 수집·보유 및 이용·제공·파기 시 보호조치와 정보주체 권리보호로 구성.

#### 침해 사고 조사 및 복구
- 사고 조사 과정 : "누가·무엇을·언제·어디서·어떻게·왜"를 결정하는 활동으로, 데이터 수집 단계와 자료 분석 단계로 나뉨.
- 네트워크 기반 침해 사고 증거 : IDS 로그, 허락받은 네트워크 모니터링 기록, ISP 가입자 이용 기록 장치/감시 장치 로그, 라우터 로그, 방화벽 로그, 인증 서버 로그를 포함.
- 디지털 포렌식 : 증거의 적법성과 무결성을 유지하면서 기술적 데이터를 획득하고 체계적으로 침해 사고를 분석하는 기술.
- 휘발성 데이터 분석 : 네트워크 접속 조사·악의적인 코드 식별 등 휘발성 데이터를 대상으로 수행하는 분석.
- 침해 사고 보고서 작성 원칙 : 데이터 획득·보관·분석 과정을 6하 원칙에 따라 명백·객관적으로 서술하고, 의사결정자가 이해하기 쉽게 디지털 포렌식 기법으로 수집한 객관적 증거를 바탕으로 작성.
- 네트워크 보안 침해 복구 조치 : 위험 우선순위 식별, 사건 본질 기술, 근원적·조직적 원인 파악, 침해 컴퓨터 복구, 밝혀진 취약점 조치(IDS·access control·firewall), 개선 책임자 지명·추적, 복구 과정·대책의 유용성 검증, 보안 정책 개선.

## 학습 4. 네트워크 보안 솔루션 업데이트 적용하기

### 4-1. 네트워크 보안 솔루션 업데이트 계획 수립

#### 패치 관리 시스템(PMS: Patch Management System)
- 개념 : 서버 등 정보 자산에 설치되지 않은 윈도우 보안 패치 등을 검색하여 자동으로 설치하는 프로그램/솔루션.
- 구성 :
  - 패치 프라이머리 서버 : 패치 파일·배포 정보를 보관·전달, 배포 서버로부터 로그 수집, 웹 기반 어드민 페이지 제공.
  - 패치 배포 서버 : 배포 대상 솔루션에 패치 파일·정보를 전달하고 로그를 수집.
- 패치 관리 시스템 보안 가이드 : APT 공격이 PMS 특징을 악용하므로(취약 시 연결된 다수 PC 감염), 초기 비밀번호 변경, 패스워드 복잡도, 중요정보 파일 미보관, 관리자 계정 최소화·임시계정 제거, 계정 공용 금지, 최소 IP·독립 네트워크 구성, 불필요 서비스·프로그램 제거, 로그 주기 확인, 공유 폴더 미사용, 이동식 매체 자동실행 차단, 최신 버전·백신 유지, 화면 보호기 10분 이내 설정 등.

---

### 4-2. 네트워크 보안 솔루션 업데이트 적용

#### 솔루션 업데이트 체계
- 개념 : 솔루션 업데이트는 배포된 SW의 기능 개선·버그·보안 취약점 수정을 위해 개발사가 내놓는 프로그램. 업데이트 체계는 최신 버전을 검색·다운로드·설치하는 체계로, 업데이트 서버·업데이트 클라이언트·통신 채널로 구성.
- 솔루션 업데이트 유형 :
  - 방식별 : 수동 업데이트(사용자가 직접 다운로드·설치), 자동 업데이트(정해진 일정에 검사·다운로드·설치). 최근에는 자동 업데이트가 일반적.
  - 중요도별 : 중요 업데이트(보안 취약점·개인정보보호·버그 수정 등 필수), 선택 업데이트(작업 환경 향상·드라이버·신규 SW 등 선택 설치).
- 동작 구조 : ① 설정정보 다운로드(최신 버전 번호·파일 URL 목록) → ② 업데이트 파일 다운로드(최신이 아니면 URL 목록으로 다운로드) → ③ 업데이트 대상 파일 설치.

#### 네트워크 보안 솔루션 백업·복구
- 특정 파일·폴더 백업 : 솔루션의 특정 파일 또는 폴더를 별도 저장장치에 백업.
- 초기(전체) 백업 : 솔루션이 설치된 장비의 출하 시 이미지 전체를 백업.
- 변경 사항 백업 : 시스템에 변경한 사항을 백업해 스냅 샷 형태로 시스템 복구 지점을 생성.
- 백업 주기 선정 : 정기적으로 백업할지, 소프트웨어·하드웨어를 추가하거나 광범위하게 수정하기 전에 백업할지 대상 유형에 따라 선택.
- 업데이트 적용 시점 : 조직의 네트워크 트래픽을 고려해 부하가 적은 시점에 적용하며, 이중화 장비는 단계적으로 패치 적용 후 이상 발생 시 즉시 중단하고 복구 조치를 수행.
