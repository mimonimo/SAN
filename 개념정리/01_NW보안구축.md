## 학습 1. NW보안 설계하기

### 1-1. NW보안요구사항 명세

#### 네트워크 보안 설계를 위한 고려 사항

- 개념 : 네트워크 보안 설계는 기관의 서버 및 클라이언트 네트워크의 내·외부에서 유발되는 사이버 침해 공격으로 인한 보안 리스크를 최소화하기 위한 것이다. 특히 개인정보 유출 방지를 위한 법률 요건이 존재하므로 법률 규제 사항의 구조에 대한 이해가 필요하다.

**1. 개인정보보호와 관련된 법률** — 개인정보보호와 관련하여 특별법과 일반법이 존재한다.
- 특별법 : 정보통신 분야의 정보통신망법, 의료 분야의 의료법, 금융 분야의 전자금융거래법 등과 같이 사업 분야별로 적용되는 법.
- 일반법 : 개인정보보호법과 같이 특별법에서 정의하지 않는 잔여 영역에 대해 포괄적으로 적용되는 법.
- 특별법과 일반법의 관계 : 충돌 시 특별법이 우선 적용되고, 특별법이 정의하지 않는 잔여 영역에는 일반법이 적용된다.

**2. 네트워크 보안 관련 법률 규제 구성** — 규제 기준으로 참조해야 하는 법은 법률·시행령·시행규칙 등으로 구성된다.
- 정보통신망법 : 정보통신망을 통해 서비스를 제공하는 사업자를 대상으로 하는 법률로, 개인정보보호 규제의 대표 법률. 이용 촉진·정보보호 규제·위법 시 벌칙 내용 포함.
- 정보통신망법 시행령 : 정보통신망법의 하위 기준으로 개인정보 보호조치를 위해 대통령령으로 제정.
- 개인정보보호법 : 특별법에 적용되지 않는 영역에 적용되는 일반법. 개인정보의 처리·보호 및 위법 시 벌칙 기준 포함.
- 개인정보보호법 시행령 : 개인정보보호법의 하위 기준으로 보호조치를 위해 대통령령으로 제정.
- 정보보호 및 개인정보보호 관리 체계 인증 등에 관한 고시 : 정보통신망법에 따른 정보보호 관리체계 인증과 개인정보보호법에 따른 개인정보보호 인증의 통합 운영에 필요한 사항을 정하는 법.

#### 네트워크 침해 공격과 관련된 주요 용어 개념

**1. 사이버 공격 관련 주요 용어**
- DDoS(디도스) 공격 : 다수의 악성코드 감염 PC를 이용해 대규모 트래픽을 특정 시스템에 전송하여 네트워크·시스템의 과부하를 일으키고 정상 서비스를 방해하는 공격.
- 스미싱 : 스마트폰 문자 메시지로 악성 앱 설치를 유도하여 개인정보·금융정보를 탈취하는 침해 기법.
- 스피어 피싱 : 창(Spear)+피싱(Phishing)의 합성어로, 특정 공격 대상을 지정해 집중적으로 공격하여 정보를 탈취하는 수법.
- 웹 해킹 : 웹서비스 요소를 공격해 이용자·관리자 권한을 획득하고 개인정보 등을 탈취하는 행위.
- 지능형 지속 공격(APT, Advanced Persistent Threat) : 특정 대상에 명확한 목표를 설정하고 지능적·지속적으로 공격하여 정보를 수집·유출하는 해킹 기법.
- 포트 스캐닝 : 공격 대상 서버의 열린 서비스 포트를 순차적으로 접속 시도하여 취약한 포트를 찾아내는 공격 준비 기법.
- 무작위 대입 공격(Brute-force) : 인증이 필요한 기능에 비밀번호 등을 무작위로 대입하여 인증 성공을 노리는 공격.
- 미라이봇넷 : IoT 기기를 통해 DDoS 공격을 하는 봇넷의 일종으로, IoT 기기의 기본 계정으로 통제권을 획득하거나 다량의 트래픽을 유발하는 공격.

**2. 정보 보안 침해 공격에 사용되는 악성 프로그램**
- 랜섬웨어 : 동의 없이 설치되어 파일을 암호화하고 복호화를 원하는 사용자에게 돈을 요구하는 악성 프로그램.
- 루트킷 : 설치된 악성코드가 백신·사용자에게 감지되지 않도록 숨겨 주는 악성 프로그램.
- C&C 서버 : 악성코드에 감염된 좀비 PC를 관리하고 DDoS 등 공격 명령을 내리는 관리·제어 목적 서버.
- 컴퓨터 바이러스 : 정상 실행 파일과 결합해 PC 동작을 중지시키거나 파일을 변조·파괴하는 악성 프로그램.
- 백도어 : 해커가 사용자 몰래 접속하여 악의적 행위를 할 수 있도록 출입구 역할을 하는 악성 프로그램.
- 좀비 PC : 원격 제어 악성 프로그램이 설치되어 사용자 모르게 스팸 발송·악성코드 유포·DDoS 등에 활용되는 컴퓨터.
- 웜(worm) : 다른 파일에 기생하지 않고 독자적으로 자신을 복제·확산하여 전파 속도가 매우 빠른 악성 프로그램(메일·공유 폴더 등으로 전파, 시스템·네트워크 부하 증가).
- 웹셸 : 해커가 원격으로 웹서버에 악의적 명령을 실행하도록 작성한 웹 브라우저 기반 스크립트 실행 파일.
- 익스플로잇(Exploit) : 소프트웨어·하드웨어 취약점을 통해 악성 행위를 수행하기 위해 제작한 프로그램.
- 제로데이 취약점 : 보안 패치가 발표되지 않은 상태에서 외부에 노출된 취약점.
- 키로거 : 사용자의 키보드 입력 데이터를 해커에게 전송하는 악성 프로그램.
- 트로이 목마 : 컴퓨터에 숨어 있으면서 사용자가 인지하지 못하도록 정보를 유출하는 악성코드.

**3. 주요 웹 해킹 공격 유형**
- SQL 인젝션 : SQL을 사용하는 응용프로그램에서 SQL 구문을 조작하여 인증 우회·권한 상승을 유발하는 공격.
- 크로스 사이트 스크립트(XSS) : 웹사이트에 악성 스크립트를 삽입하는 취약점을 활용해 의도하지 않은 기능 수행·정보 탈취를 하는 공격.
- 크로스 사이트 요청 위조(CSRF) : 피해자의 세션 쿠키·인증 정보를 자동 포함시켜 위조된 웹 요청을 강제로 보내, 웹서버가 정당한 요청으로 오해하게 만드는 공격.
- 파일 업로드(웹셸) : 허용되지 않은 유형의 파일을 업로드하고 외부에서 직접 접근하는 취약점을 활용해 웹서버 권한을 탈취하는 공격.
- 파일 다운로드 : 검증되지 않은 외부 입력값으로 서버 파일에 접근하여 중요 서버 파일을 탈취하는 공격.
- 세션 탈취 : XSS 등으로 피해자의 세션 쿠키를 탈취하고 이를 이용해 피해자 세션으로 로그인하는 공격.

#### ISMS-P 인증 기준 중 네트워크 보안 관련 항목 (참고)
- 네트워크 접근(2.6.1) : 비인가 접근 통제를 위해 IP 관리·단말 인증 등 절차를 수립하고, 업무 목적·중요도에 따라 네트워크 분리(DMZ, 서버팜, DB존, 개발존 등)와 접근 통제를 적용.
- 정보시스템 접근(2.6.2) : 서버·네트워크시스템 등 정보시스템에 접근 허용 사용자, 접근 제한 방식, 안전한 접근 수단 등을 정의·통제.
- 무선 네트워크 접근(2.6.5) : 사용자 인증, 송수신 데이터 암호화, AP 통제 등 무선 보호 대책 적용 및 비인가 AP·AD Hoc 접속 차단.
- 원격 접근 통제(2.6.6) : 보호구역 외 정보시스템 관리·개인정보처리는 원칙 금지, 불가피 시 책임자 승인·단말 지정·강화된 인증·구간 암호화 등 보호 대책 수립.
- 인터넷 접속 통제(2.6.7) : 정보 유출·악성코드 감염·내부망 침투 예방을 위해 주요 시스템·단말의 인터넷 접속/서비스(P2P·웹하드·메신저 등)를 제한.

#### 네트워크 보안요구사항 도출 항목
- 서버 네트워크 보안요구사항 : 접속 권한 IP 제한, 불법 개인정보 유출 시도 탐지, 망분리, 외부망 접속 시 VPN·전용선 등 안전 접속, 원격관리 원칙 금지, 중요도별 내·외부망 분리, DDoS 방어, 악성코드 유입 차단, 개인정보 유출 탐지·차단.
- 클라이언트 네트워크 보안요구사항 : 허가받지 않은 단말 접속 차단, 내부망 관리 시 특정 단말 제한, 무선랜 사용 시 사용자 인증·송수신 암호화 등 보호 대책 수립.

### 1-2. NW보안 설계

#### 네트워크 보안 설계 원칙 및 보안 장비 유형

**1. 네트워크 보안 설계 원칙**
- 최소화의 원칙(Minimalization) : 반드시 필요한 항목을 필요로 하는 사람에게 최소한의 기간 동안만 접근을 허용하는 것. 네트워크 접근 통제의 기본 원칙.
- 분리의 원칙(Isolation) : 리스크를 격리하기 위한 원칙. DMZ와 내부망 분리, 개발 서버와 운영 서버 분리, 업무 처리자와 관리자의 권한 분리 등이 예.
- 다중 보안의 원칙(Defence in Depth) : 하나의 보안대책이 무력화되더라도 다른 보안대책으로 전체 보안성이 확보되도록 하는 개념(예: 침입 방지(1차)→확산 침투 방지(2차)→외부 유출 방지(3차)의 3중 대책).
- 법규 준수의 원칙(Compliance) : 보안 리스크 감소와 법률 위반 방지를 위해 법규가 가이드하는 보안대책을 반드시 적용.

**2. 네트워크 보안 장비 유형** — 접근 통제, 침입 차단·탐지, DDoS 탐지 등을 수행하는 장비.
- 방화벽(Fire Wall) : 외부↔내부 정보통신망에 불법 접근하는 것을 차단하는 시스템. 발신지·수신지 IP 주소와 포트 번호에 대한 접근 통제 수행.
- 침입탐지시스템(IDS) : 네트워크를 통한 사이버 공격을 실시간으로 탐지.
- 침입예방시스템(IPS) : 네트워크를 통한 사이버 공격을 실시간으로 탐지하고 차단.
- DDoS방지시스템 : 목적지 기준 트래픽이 임계치를 초과하면 기존 세션을 제외한 패킷을 Null Routing 처리하여 DDoS에 대응.
- 웹방화벽 : 알려진 유형의 웹 해킹 공격을 탐지·방어.

> 방화벽 주요 기능: 패킷 필터(L3·L4), 상태 확인(L4), 주소 변환 NAT·DHCP(L4), 트래픽 관리(L4), 프록시(L3·L4), 콘텐츠 필터(L7), URL 필터(L7), 바이러스 필터(L7), MAC 바인딩(L2·L3), 영역 분리(L3), 운영 모드 Router/Bridge(L2·L3).
> IDS vs IPS: IDS는 애플리케이션 계층·미러링·탭 모드·라우터 트래픽 탐지·간접 차단(방화벽 연동), IPS는 커널 계층·후킹·인라인 모드·로컬 트래픽 탐지·패킷 드롭(직접 차단). 탐지 성능은 IPS가 빠르나 장애 시 로컬 네트워크 장애 발생.

#### 망분리의 개념 및 유형

**1. 망분리의 개념 및 근거**
- 망분리의 개념 : 시스템의 오용/악용으로부터 정보시스템을 보호하기 위해, 중요 내부 업무 처리자의 PC·노트북을 인터넷에 접속되지 않도록 차단하는 기술.
- 망분리의 근거 : 개인정보의 기술적·관리적 보호조치 기준 제4조(접근 통제) 6항 — 일정 규모 이상(이용자 수 일평균 100만 명 이상 또는 매출 100억 원 이상)의 정보통신서비스 제공자는 개인정보취급자 컴퓨터를 물리적·논리적으로 망분리해야 함.

**2. 망분리의 유형 및 구현 방식**
- 물리적 망분리 : 2대 이상의 PC와 별도 회선으로 내부망·외부망을 분리하는 방식(2PC, 공용 PC 방식).
- 논리적 망분리 : 한 대의 PC에서 가상화 기술로 내부망·외부망을 분리하는 방식(서버 기반 가상화, Host 기반 가상화).

### 1-3. NW구현 준비

#### 정보통신 네트워크 구축 절차
- 개념 : 네트워크 구축 절차는 기본적으로 4단계로 나뉘며, 순환 주기는 소요 트래픽 증가량·서비스 환경 변화에 따라 달라진다.
- 1단계 진단 및 분석(Analysis) : 제공 서비스 요구사항 분석, 기존 네트워크 기술적 분석으로 기본 품질 목표 결정, 제약사항에 따른 최종 품질 목표 도출.
- 2단계 설계(Design) : 품질 목표 만족을 위한 구축 목표·원칙 수립, 아키텍처 설계 및 구간별 장비 용량 산정, 장비·솔루션 선정, 구축 일정 계획 수립.
- 3단계 설치/구축(Implementation) : 설계·일정 계획에 따른 장비 구축, 결과가 요구사항·품질 목표를 만족하는지 검증.
- 4단계 운용(Operation) : 운용 수행, 품질 기준 목표 유지 관리, 차기 고도화 진단 분석의 기초 자료로 활용할 운용 품질 지표 수집·관리.

#### 정보통신 네트워크 설계 개요 및 세부 요건

**1. 네트워크 설계 개요**
- 네트워크 설계 업무 : 신규 구축·장비 대체 및 증설·구조 변경 등에 대해 계획을 세우고 설계 도면·내역서 등을 작성하는 과정.
- 네트워크 설계 목적 : 노드 간 상호 통신이라는 기능적 요구사항과 가용성·속도·저지연 등 품질 관련 비기능적 요구사항을 충족하기 위한 것.

**2. 네트워크 설계의 요건**
- 경제성 : 투자 비용 대비 정량적·정성적 효과를 고려.
- 가용성 : 일부 노드 장애에도 전체 서비스 중단이 없도록 이중화 등을 고려.
- 확장성 : 구축 이후 장비·운용 관리 시스템의 확장·변경이 용이하도록 기술 호환성을 고려.
- 운용 효율성 : 운영 편이성·업무 효율성·대응 용이성 등 운영관리 효율성을 고려.

## 학습 2. NW보안 구현하기

### 2-1. NW보안 구현

#### 네트워크 ACL 설정

**1. 네트워크 ACL의 개념**
- 네트워크 ACL의 정의 : 허가받지 않은 이용자·트래픽이 라우터나 네트워크의 특정 자원에 접근하는 것을 차단·허용하는 것. 라우터 인터페이스에 적용되어 사전 설정 내역에 따라 특정 패킷을 필터링.
- ACL 적용 장점 : 불필요한 트래픽을 줄여 네트워크 부하 감소 및 보안성 향상.
- ACL 적용 단점 : 패킷 검사 과정에서 지연 발생, 과다 적용 시 라우팅 성능 저하.

**2. 네트워크 ACL의 종류**
- 표준 ACL(Standard ACL) : 패킷의 출발지 주소만 검사하여 제어. ACL 번호 1~99, 1300~1699.
- 확장 ACL(Extended ACL) : 출발지·목적지 주소, 프로토콜, 포트 번호 등을 검사하여 제어. ACL 번호 100~199, 2000~2699.

#### 보안 운영관리를 위한 ESM

**1. ESM(Enterprise Security Management) 개요**
- ESM의 개념 : 네트워크에 유입되는 다양한 위험 요소를 총체적으로 분석하여 미리 예방할 수 있도록 지원하는 솔루션.
- 주요 기능 : 보안 이벤트 수집·가공(정규화·필터링·축약), 통합 관리(단일/이종 장비 연계 분석, 실시간 모니터링), 침해 경보(수준별 가시·가청·메일·메시징 경보), 침해 대응(자동/수동 대응, 증적·로그 보관).

**2. ESM의 구성** — 이벤트 수집 에이전트, 매니저, 관리 콘솔로 구성.
- ESM 에이전트 : 보안·시스템·네트워크 장비에 탑재되어 정의된 규칙에 따라 이벤트를 수집·전달하거나 보안 정책을 반영.
- ESM 매니저 : 정의된 규칙으로 이벤트 데이터를 분석·저장하고 정제된 이벤트를 콘솔로 전달.
- ESM 콘솔 : 전달된 자료를 시각화하여 운영자 판단을 지원하고, 침해 발생 시 경보 발령 및 자동/수동 대응 지원.

**3. ESM 기반의 보안관제 업무**
- 보안관제의 정의 : 외부 위협의 침입 가능 경로를 모니터링하고 이벤트를 감지하여 침해사고를 예방하는 활동.
- 보안관제의 주요 역할 : 보안시스템 통합 관리, 일관성 있는 정책 구현, 신속한 대응 처리(24×365 실시간 감시).
- 보안관제의 단계별 활동 : 예방 단계(보안 취약점 패치, 취약점 점검, 정책 관리) → 탐지 단계(모니터링, NMS·Alert, 시스템 장애·관리적 이벤트) → 대응 단계(웜·바이러스, 스캐닝, 침해사고, 기타 사고 대응).

#### 네트워크 장비 보안 관리
- 필요성 : 스위치·라우터·VPN·방화벽 등 네트워크 장비도 O/S를 운영하므로 서버처럼 보안 관리가 필요.
- 네트워크 계정 관리 : 관리 콘솔 접속 시 기본 패스워드 변경, 패스워드 암호화 적용, 계정별 권한 수준 설정 점검.
- 네트워크 접근 관리 : 원격 접속(VTY) 시 ACL로 허용 IP 통제, 접속 타임아웃 설정, 불필요한 보조(AUX) 포트 비활성화, 로그온 경고 메시지 출력 점검.

### 2-2. NW보안 테스트

#### 테스트 개요

**1. 테스트의 일반적인 절차**
- 테스트 계획 : 요구사항 수집·계획 수립·계획서 작성, 항목별 테스트 케이스 작성 및 테스트 데이터 확보.
- 테스트 수행 : 테스트 환경 구축·실행·측정, 결과 분석으로 결함 도출 및 테스트 결과서 작성·보고.
- 결함 관리 : 결함 조치 방법·일정이 포함된 계획 수립(핵심 결함과 그 외 결함 구분), 결함 수정 후 재테스트 및 이행 관리.

**2. 테스트 계획 수립 단계의 세부 업무** : 테스트 범위 정의, 착수·완료 기준 정의, 테스트 환경 및 소요 자원 정의, 일정 정의, 기타 사항(고려 사항·위험 요소·결함 관리 방안) 정의, 테스트 계획서 작성·검토·승인.

#### 테스트 유형 및 점검 기준
- 단위 테스트 : 단위 모듈 또는 네트워크 단일 노드 내 기능이 요구사항에 부합하는지 테스트(인터페이스·자료 구조·수행 경로·오류 처리·경계 테스트).
- 통합 테스트 : 단위 테스트 기반으로 모듈 간·노드 간 연계 기능을 테스트.
  - 하향식 테스트 : 최상위 모듈부터 하위로 통합(깊이 우선/넓이 우선 가능).
  - 상향식 테스트 : 최하위 모듈부터 상위로 통합.
  - 혼합식 테스트 : 하향식·상향식을 절충, 중요 모듈 중심으로 통합.
  - 비점진적 테스트 : 모든 모듈을 한꺼번에 통합하여 테스트.
- 시스템 테스트 : 통합 테스트 기반으로 실제 인프라에서 기능 검증 및 비기능(처리 속도·안정성) 요구사항 검증. 검증 항목 — 기능성·신뢰성·사용성·효율성·유지보수성·이식성.
- 사용자 인수 테스트 : 구현 기능이 최종 사용자의 업무 완결성 관점에서 요구사항에 부합하는지 테스트.
  - 알파 테스트 : 사용자 테스트를 개발자 환경에서 통제된 상태로 수행.
  - 베타 테스트 : 개발자가 참여하지 않고 일정 수의 시범 사용자가 수행.
